Why this page exists

Automation, for me, is not only about scripts. It is about turning repeated actions into structured workflows, making technical processes easier to understand, easier to repeat and easier to improve.

My workflow

Each project follows the same idea: take something hidden in code, desktop apps, scripts or infrastructure notes — then turn it into something visible, explainable and easy to review.

1
Identify the system Find the repeated process, workflow, support task or deployment pattern.
2
Structure the logic Break the project into inputs, process steps, output and documentation.
3
Make it visible Deploy a live preview, dashboard, simulation or case study layer.
4
Connect it to the portfolio Link live demos, GitHub repositories and clear project context in one place.
